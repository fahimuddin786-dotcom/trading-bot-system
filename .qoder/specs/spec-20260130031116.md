# Fix Render Build Failure - Trading Bot System

## Problem
Build command fails on Render with error: `sh: 1: cd: can't cd to ../frontend`

**Root Cause:** The `install:all` and `build` scripts in `backend/package.json` use `cd ../frontend`, which assumes execution from the `backend/` directory. If Render runs from the repo root, these paths break.

## Current Structure
```
trading-bot-system/
├── backend/
│   └── package.json  (contains install:all, build scripts)
├── frontend/
│   └── package.json
└── (no root package.json)
```

## Recommended Solution: Update backend/package.json scripts

Modify the scripts to work correctly regardless of Render's root directory configuration:

### File: `backend/package.json`

**Current scripts:**
```json
"install:all": "npm install && cd ../frontend && npm install",
"build": "cd ../frontend && npm run build",
```

**Updated scripts:**
```json
"install:all": "npm install && npm install --prefix ../frontend",
"build": "npm run build --prefix ../frontend",
```

Using `--prefix` is more reliable than `cd` because:
- Works from any directory
- Doesn't depend on shell behavior
- More explicit about target directory

## Alternative: Configure Render Dashboard

If you prefer not to change code, set these in Render's dashboard:
- **Root Directory:** `backend`
- This makes the existing `cd ../frontend` paths work
